# Lab 3: Network Traffic Analysis with Wireshark/TShark

  ## Environment
  - OS: Kali Linux (WSL on Windows 10)
  - Tool: TShark 4.4.6 (CLI Wireshark)
  - Interface: eth0
  - Date: 2026-04-06

  ## Objective
  Capture and analyze live network traffic to identify protocols,
  understand normal network behavior, and spot potential security
  observations using TShark.

  ## Setup
  Install Wireshark/TShark on Kali:
  ```bash
  sudo apt update && sudo apt install -y wireshark
  sudo usermod -aG wireshark $USER
  newgrp wireshark

  Capture

  Captured 200 packets on eth0 and saved to pcap file:
  tshark -i eth0 -c 200 -w ~/lab3_capture.pcap
  Generated ICMP traffic to populate the capture:
  ping -c 50 google.com

  ICMP (98 frames)

  MDNS (67 frames)

  Multicast DNS traffic revealing devices on the local network.
  A smart home device was identified passively via MDNS broadcasts.

  Security Note: MDNS enables passive device reconnaissance
  without active scanning. Attackers can identify IoT devices
  on a network silently.

  ARP (7 frames)

  Address Resolution Protocol mapping IPs to MAC addresses.
  Example packet:
  Who has [GATEWAY-IP-REDACTED]? Tell [HOST-IP-REDACTED]
  No ARP spoofing detected. Normal baseline ARP activity.

  Security Note: ARP has no authentication — vulnerable to
  ARP spoofing/poisoning attacks (MITM).

  Key Takeaways

  1. Passive network capture can reveal devices without active scanning
  2. ARP is unauthenticated and a common MITM attack vector
  3. DNS query analysis is a fast way to spot suspicious outbound traffic
  4. Clean captures establish a baseline — anomalies stand out against it

  OWASP LLM Relevance

  This lab supports understanding of LLM06: Excessive Agency —
  an LLM-powered agent making unexpected network connections would
  appear in DNS/TCP traffic and could be caught with this technique.

  Commands Reference

  # Capture 200 packets
  tshark -i eth0 -c 200 -w ~/lab3_capture.pcap

  # Read capture file
  tshark -r ~/lab3_capture.pcap

  # Filter by protocol
  tshark -r ~/lab3_capture.pcap -Y "icmp"
  tshark -r ~/lab3_capture.pcap -Y "dns"
  tshark -r ~/lab3_capture.pcap -Y "arp"

  # Protocol hierarchy summary
  tshark -r ~/lab3_capture.pcap -q -z io,phs

  # Extract DNS queries
  tshark -r ~/lab3_capture.pcap -Y "dns" -T fields -e dns.qry.name
