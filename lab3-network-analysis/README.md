## Objective

Capture and analyze live network traffic to identify protocols, understand normal network behavior, and spot potential security observations using TShark.

---

## Setup

Install Wireshark/TShark on Kali:

```bash
sudo apt update && sudo apt install -y wireshark
sudo usermod -aG wireshark $USER
newgrp wireshark
```

---

## Packet Capture

Captured 200 packets on `eth0` and saved to a pcap file:

```bash
tshark -i eth0 -c 200 -w ~/lab3_capture.pcap
```

Generated ICMP traffic to populate the capture:

```bash
ping -c 50 google.com
```

## Protocol Analysis

### ICMP (98 frames)

Ping request/reply pairs generated by `ping google.com`. Normal traffic — no anomalies detected.

### MDNS (67 frames)

Multicast DNS traffic revealing devices on the local network. A smart home device was identified passively via MDNS broadcasts.

> **Security Note:** MDNS enables passive device reconnaissance without active scanning. Attackers can silently identify IoT devices on a network.

### ARP (7 frames)

Address Resolution Protocol mapping IPs to MAC addresses.

```
Example: Who has [GATEWAY-IP-REDACTED]? Tell [HOST-IP-REDACTED]
```

No ARP spoofing detected. Normal baseline ARP activity.

> **Security Note:** ARP has no authentication — vulnerable to ARP spoofing/poisoning attacks (MITM).

### DNS (6 frames)

Only legitimate queries to `google.com` and a reverse lookup on a public IP. No suspicious outbound DNS queries detected.

### ICMPv6 (22 frames)

IPv6 neighbor discovery traffic. Normal behavior for a dual-stack network.

| Finding | Severity | Notes |
|---------|----------|-------|
| Smart home device visible via MDNS | Low | Passive recon possible on LAN |

| DNS queries clean | Info | No suspicious domains |
| No unexpected TCP connections | Info | No signs of exfil or C2 traffic |

---

## Key Takeaways

1. **Passive capture reveals devices** — Network capture can identify hosts without active scanning.
2. **ARP is unauthenticated** — A common vector for MITM attacks via ARP spoofing/poisoning.
3. **DNS query analysis** — A fast way to spot suspicious outbound traffic and potential C2 beaconing.
4. **Baselines matter** — Clean captures establish a normal baseline; anomalies stand out against it.

## Commands Reference

```bash
# Capture 200 packets to file
tshark -i eth0 -c 200 -w ~/lab3_capture.pcap

# Read a capture file
tshark -r ~/lab3_capture.pcap

# Filter by protocol
tshark -r ~/lab3_capture.pcap -Y "icmp"
tshark -r ~/lab3_capture.pcap -Y "dns"
tshark -r ~/lab3_capture.pcap -Y "arp"

# Protocol hierarchy summary
tshark -r ~/lab3_capture.pcap -q -z io,phs

# Extract DNS query names
tshark -r ~/lab3_capture.pcap -Y "dns" -T fields -e dns.qry.name
```
